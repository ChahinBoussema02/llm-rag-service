# Incident Response (AcmeAI)

## Definitions
- "Service unavailable" means API and web app are inaccessible for most users.
- An incident is "confirmed" when on-call has validated it and posted a status update.

## Process
1. Detect and triage
2. Mitigate impact
3. Communicate status updates
4. Post-incident review (PIR) within 5 business days

## Refund-related Note
- If a confirmed incident causes service unavailability longer than 24 hours, partial refunds may be issued per the Refund Policy.
